
Инкапсуляция один из ключевых принципов ООП. Это механизм сокрытия внутренего состояния обьекта и предотавления доступа к нему через специальные безопасные интерфейсы.

- В Python реализован иначе нежели в других языках, тут приватность работает путем сокрытия и согласования, а не блокирования доступа. Мы можем получить доступ, но это будет плохой практикой. Используем геттеры и сеттеры и декоратор @property для доступа к приватным аттрибутам и методам.

- Помогает валидировать данные
- Защищает от поломки
- Позволяет менять внутренее устройство не изменяя внешнее.

Существует модуль [[accesify]] который позволяет реализовывать инкапсуляцию как в C языках. 




Приватность:
	`_name`  - защищенный. Просто предупреждение 
	`__name` - приватный. Меняет название с помощью name mangling на `_class.__name`
	`@property `- декоратор для контроля доступа

**Применение**:

**Пример**:
```
class User:
    def __init__(self, age):
        self.__age = age  # приватный атрибут

    @property
    def age(self):
        return self.__age

    @age.setter
    def age(self, value):
        if value < 0:
            raise ValueError("Возраст не может быть отрицательным")
        self.__age = value

```
