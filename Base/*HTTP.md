Является основным протоколом общения между приложениями по сети

- Методы
	- GET - метод получения ресурса, запросы могут только извлекать данные.
	- POST - метода для добавления сущностей к ресурсу. Вызывает изменение состояния и побочные эффекты на сервере.
	- HEAD - метод запрашивает ресурс без тела ответа
	- PUT - метод полной замены ресурса данными запроса
	- DELETE - метод удаления ресурса
	- CONNECT - метод устанвки тунеля к серверу или определенному ресурсу
	- OPTIONS -  метод для описания параметов соединения с ресурсом
	- TRACE - метод возвращения тестового сообщения с ресурса
	- PATCH -метод частичного изменения ресурса
- Статус коды 
	-  200 - OK
	- 100 - Info
	- 300 - Redirect
	- 400 - Client Error
	- 500 - Server Error
