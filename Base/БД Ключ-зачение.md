Хранят данные как совокупность пар ключ-значение. Где ключ - уникальный идентификатор.



![[key_value_dbms.webp]]

- Преимущества
	- Скорость работы. Хещ таблица А значит скорость O(1).
	- Простота. Нет сложных схем как в реляционных базах данных.
	- Гибкость: Может включать JSON
	- Масштабируемость. Легко масштабировать по горзонтали
- Недостатки
	- Неэффективен при работе с группой записи.
		- Нет гарантии того что фактически получаем по ключу и нужно фильтровать и обрабатывать данные.
	- Отсутствие языка запроса. 
		- Логика будет вшита в код, что может усложнить поддержку кода.
