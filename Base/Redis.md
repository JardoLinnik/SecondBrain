Redis - сверхбыстрая база данных, хранящая данные в RAM. Имеется возможность хранить и на диске. Часто называется In-memory data store

База данных которая применяется как Кеш и иногда как [[Брокер сообщений]]

Преимущества:
- Скорость работы - работает из оперативной памяти.
- Поддерживает различные структуры данных
- Встроенный механизм обмена сообщениями. 
	- Pub/Sub
- TTL. Можно задать срок жизни ключа для минимизации перенагрузки
- Масштабирование. Репликация и кластеры
- Простота. Простые команды


Применение: 
1. Кеш
2. Сессии пользователей
3. Очередь задач
4. Рейтинг, лайки, счетчик
5. Чат, уведомление, стриминг
6. Хранение временных данных

**Пример**:
1. Хранение данных. Чтобы не обращаться к медленной базе. В e-commerce кешируются карточки товаров или корзина пользователя. 
2. Хранение сессий для быстрой идентификации пользователей
3. Используют в виде очередей в Celery или RQ
4. Благодяря INCR и Sorted Sets можно реализовать систему рейтингов.
5. Может быть брокером сообщений для чатов или систем стриминга.
6. Одноразовые токены, коды подтверждения(ОТР), данные с ограниченным временем жизни.
