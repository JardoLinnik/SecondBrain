Использование нескольких процессов для запуска задач параллельно. 

Многопроцессность более тяжелая по ресурсам чем многопоточность, но имеет меньшую зависимость и если в одном процессе произойдет сбой, то остальные продолжат работу.

- Модули для работы с процессами
	- Модуль [[*os]]
	- Модуль [[**multiprocessing]]
- Недостатки 
	- Задействует много ресурсов, от памяти - выделение отдельных ресурсов на процесс, до скорости погружение всех модулей и интерпритатора в новый процесс.
	- Создане механизмов для общения между двумя процессами(костыли)
- Блокировка процессов
	- Семафор
	- [[Контекстный менеджер]]
	

**Проблема**: Возможность эффективно использовать многоядерные процессоры.

**Пример**: Многопроцесность можно сравнить с еще одним работником, который берет на себя часть обязнностей.

**Доп. ссылки:** [[Процесс]] [[* Паралельное програмирование]] [[*Многопоточность]], [[Семафор]]

Практика:

**Через контекстный менеджер:**
```
def worker(lock, shared_resource):
    with lock:
        shared_resource.value += 1
        print(f"Процесс {shared_resource.value}")

```


**Явный метод блокировки:**
```
def worker(lock, shared_resource):
    lock.acquire()
    try:
        shared_resource.value += 1
        print(f"Процесс {shared_resource.value}")
    finally:
        lock.release()

```
