В компании вкусвилл я непосредственно занимался созданием движка мультиканальной синхронизации товарных остатков в реаьном времени и антифрод сервисом.

Задача была в том чтобы собрать данные из разных источников (ERP, WMS, дарксторов) и предосавить в единую консистентную витрину товарных остатков. Это позволяелоизбежать ситуации когда клиент видит товар а по факту его нет.

Мы реализовывали систему на базе микросервисов.

Каждый источник имел свой сервис конектор он подписывался на изменения в источнке и транслировал события в kafka. - 3 сервиса.

Агрегационный сервис Собирающий данные из потока kafka собирал данные и писал их в postgreSQL + Redis.

Api витрина - отдавал готовую витрину наружу.
