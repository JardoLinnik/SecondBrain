Время отклика. Время за которое пользователь послал запрос и получил ответ.
Для **скорости работы бизнеса/сервиса** → важен response time.

- Состав
	-  [[Latency]] + время обработки + время передачи результата

- Высокая response time
	- Cистема может быстро "откликнуться" (низкая latency), но слишком долго готовить результат (например, тяжёлый SQL-запрос или плохая оптимизация алгоритма).

- **Фокус на response time**
	- Критично для **batch-задач**, API-запросов, поиска, e-commerce.
	- Если страница загружается 10 секунд — пользователю всё равно, что сервер начал отвечать через 10 мс, результат плохой.
	- Оптимизация: кеширование, оптимизация БД, параллелизм, балансировка нагрузки, масштабирование сервисов.
