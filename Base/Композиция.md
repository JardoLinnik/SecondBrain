Композиция — это когда один объект включает другой объект внутри себя и использует его. Паттерн.

Не наследуешься, а собираешь обьект из других обьектов.

### Легче тестировать
Ты можешь подставить подменный объект:
`order = Order(email_service=FakeEmailService())`
###  Легче расширять
Можно заменить часть без переписывания структуры класса.
###  Не ломается MRO и нет конфликтов имён
Композиция проще и безопаснее.
###  Лучше для код-стайла
В продакшене избегают глубоких наследований.


**Применение**:

Наследуйся только когда “является”.  
Во всех остальных случаях — используй композицию.

**Пример**:
```
class Engine:
    def start(self):
        return "Engine started"

class Car:
    def __init__(self):
        self.engine = Engine()  # Car имеет Engine

    def go(self):
        return self.engine.start()

```
