GIL — это глобальная блокировка в CPython, которая гарантирует, что одновременно исполняется только один поток байткода. Она нужна для упрощения работы интерпретатора и безопасного управления памятью.

Основная проблема в том, что многопоточность в Python не масштабируется на несколько ядер — CPU-bound задачи всё равно выполняются последовательно. Для обхода используют multiprocessing или async I/O.

- Для чего вообще нужен?
	- Упрощение реализации интепритатора CPython:
		- управление памятью, сборщик мусора (GC) и работа с объектами становятся потокобезопасными.
	- Без GIL пришлось бы оборачивать все операции с объектами Python в локи, что сильно замедлило бы работу.

- Какие проблемы вызывает?
	- **Многопоточность ограничена**
    - потоки в Python не могут реально выполняться параллельно на нескольких ядрах,
    - для CPU-интенсивных задач многопоточность бесполезна (даёт даже оверхед).
	- **Неравномерное распределение времени**
	    - GIL переключает потоки через квант времени → возможен “thrashing”, когда много потоков дерутся за GIL.
	- **Блокировка при I/O и heavy CPU mix**
	    - если один поток в тяжёлых вычислениях держит GIL, остальные «висят».
