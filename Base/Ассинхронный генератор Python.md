Это обычный генератор  с таким же ключевым словом yeald возвращающий значения по мере итерации, но работающий в ассинхронном контексте. В отличие от обычного генератора  не передает управление event loop а следовательно не блокирует нам поток до получения следующего значения.

Тоесть когда задаем функцию генератора вызываем с `async def`
Вызывать его можно через async for 

Тем самым это позволяет производить потепенные манипуляции с данными  при этом внутри можно использовать await - ходить в сеть, ждать ответ или обрабатывать поток сообщений.


**Собес**: Я применял их когда нужно было работать с потоком данных при чтении событий с kafka, оборачивал это все в ассинхронный генератор и тем самым постепенно читал события путем `async for`

При работе с API чтобы клиенту не отдавать сразу большой обьем данных это все тоже оборачивалось в ассинхронный генератор и через `StreamingResponse`в FastApi  клиент получал данные по мере готовности, а не сразу весь ответ. 

**Пример**:

```
async def async_gen():
    for i in range(5):
        await asyncio.sleep(1)
        yield i



async for x in async_gen():
    print(x)



```
