Реализация взаимодействия общения между сервисами взамен HTTP. 
Брокер сообщений - промежуточное звено, где сообщения сохраняются и далее отсылаются тем сервисам которым нужно.

Producer - производитель сообщений.
Consumer - потребитель сообщений.

Плюсы брокера сообщений:
- Ассинхронное взаимодействие. Как только данные в брокере Producer забывает об этой задаче и выполняет следующие.
- Нет потери сообщения даже при неполадках с Consumer
- Масштабируемость. Даже если есть несколько Consumer, то каждый из них может спокойно получить такие же данные через брокер, не затрагивая логику Producer


- Проблемы HTTP:
	- Нет гарантии доставки данных по интернету .
	- Большая нагруженность кода при отправке данных нескольким микросервисам. Потенциальные баги.
	- Масштабируемость.

Брокеры сообщений:
- [[Kafka]]
- RabbitMQ
- Redis
- ActiveMQ
