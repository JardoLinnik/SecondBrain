Полиморфизм - это один из ключевых принципов ООП. Один и тот же метод можно отнести к разным обьектам. и каждый выполнит его по-своему.

Делает код 
- Гибким
- Расширяемым
- Чистым
- Поддерживаемым

# Типы полиморфизма в Python

### 1) **Полиморфизм наследования**
Переопределяешь методы в дочерних классах.

### 2) **“Утиная типизация” (duck typing)**
Python не требует наследования — важно лишь, что метод существует.
> “Если что-то крякает как утка и ходит как утка — значит, это утка.”


**Применение**:

1. Разные обработчики запросов
2. Полиморфные селиализаторы
3. Разные способы ханения данных
4. Платежные системы
5. ORM
6. Middleware
7. Логгеры и хэндлеры

**Пример**:
```
class EmailSender:
    def send(self, message):
        print("Send email:", message)

class SmsSender:
    def send(self, message):
        print("Send SMS:", message)

```
