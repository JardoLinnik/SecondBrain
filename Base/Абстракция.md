Абстракция — это принцип ООП, который позволяет скрывать сложность и показывать только важную часть.

В продакшене часто используют **абстрактные классы** — шаблоны для будущих классов.

```
from abc import ABC, abstractmethod

class PaymentSystem(ABC):

    @abstractmethod
    def pay(self, amount):
        pass

```

`

А потом конкретные реализации:

```
class PayPal(PaymentSystem):
    def pay(self, amount):
        print(f"PayPal: {amount}")

class Stripe(PaymentSystem):
    def pay(self, amount):
        print(f"Stripe: {amount}")

```

Твой бекенд работает с **абстракцией**, не зная деталей:

```
def process_payment(gateway: PaymentSystem, amount):
    gateway.pay(amount)

```
Ты передаёшь хоть Stripe, хоть PayPal — функция работает.


**Применение**:
Скрыть детали реализации и дать простой интерфейс.

**Если абстракция может использоваться разными частями проекта — выноси в отдельный модуль.**  
Если она нужна только внутри класса — можешь оставить рядом.
