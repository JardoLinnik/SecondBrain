Git - система контроля версии нужный для отката изменений и сохранения нескольких версий проекта. При изменении проекта сохраняет внутри себя нестабильную версию проекта с внесенными изменениями ,а также версии других разработчиков для дальнейшей компиляции.


```
#задать имя и почту пользователя
git config --global user.name "..."
git config --global user.email  "..."
```

[[GitHub]] - веб-приложения для предоставления своих проектов и проектов других разработчиков выставляемых как приватно так и в открытый доступ на облако.

Репозиторий - директория (папка) которая привязана к Git. Локальный репозиторий это директория на самом устройстве , а удаленный хранится в облаке ,но основан на локальном. В папке локального репозитория создается скртый фаил .git.

```
#создание локального репозитория
git init 
#создание удаленного репозитория
git remote add origin "URL"
```

Git сам по себе только следит за изменениями внутри репозитория, а для отправки данных на GitHub нужно будет сделать все вручную.
\
```
#Копировать репозиторий к себе на устройство
git clone "URL"
```

Коммит - сохранение изменений версии внутри git. После коммита присваиваеться номер к версии - хеш коммита. 
Индекс Git (staging area) - пространство куда git запоминает и подготавливает файлы к отправке в коммит.


```
# показывает измекнения и файлы которые готовы к коммиту
git status 
#добавить в коммит все или какие то обьекты
git add . 
git add /file_name
# добавить в коммит
git commit -m "message"
```

```
#Увидеть все локальные коммиты
git log
#Возвращение к окальному коммиту
git chekout "хеш коммита"
```

```
#загрузить ветку на удаленный репозитоории
git push origin master
#скачать все измененные файлы с удаленного репозитория
git pull origin master
```

Ветка GIT - сохранение разных версий приложения и нескольких коммитов внутри  репозитория.

```
#просмотр всех веток
git branch
#удаление ветки
git branch -D ""
#создать ветку
git branch "имя ветки"
#переключится на другую ветку
git checkout "имя ветки"
#создать и сразу переключится на ветку
git checkout -b ""
```

Слияние - объединение веток к главной ветке.

```
#слияние путем переноса всех изменений в один коммит
git merge 'имя ветки которую хотим обьеденить с главной'
#слияние путем сохранения коммитов ветки и переноса их в мастер
git rebase 'имя ветки'
```

```
#фаил в которые можно вносить файлы не отслеживаемые git
.gitignore
```
