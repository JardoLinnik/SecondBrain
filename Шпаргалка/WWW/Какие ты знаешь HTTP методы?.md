* GET:  
- Описание: Запросить представление ресурса.  
- Особенности: Безопасный и идемпотентный метод (не изменяет состояние сервера).  
* POST:  
- Описание: Отправить данные для создания нового ресурса.  
- Особенности: Неидемпотентный (многократное выполнение может привести к созданию нескольких ресурсов).  
* PUT:  
- Описание: Обновить существующий ресурс или создать новый, если он не существует.  
- Особенности: Идемпотентный.  
* DELETE:  
- Описание: Удалить указанный ресурс.  
- Особенности: Идемпотентный.  
* HEAD:  
- Описание: Получить заголовки ответа без тела.  
- Особенности: Полезен для проверки доступности ресурса или получения метаданных.  
* OPTIONS:  
- Описание: Описать доступные методы для указанного ресурса.  
- Особенности: Используется для CORS (Cross-Origin Resource Sharing) и проверки возможностей сервера.  
* PATCH:  
- Описание: Частично обновить ресурс.  
- Особенности: Неидемпотентный
