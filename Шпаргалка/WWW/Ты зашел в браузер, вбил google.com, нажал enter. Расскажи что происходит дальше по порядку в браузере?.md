1. Браузер валидирует адрес (правильно ли указан протокол и остальные части адреса)  
2. Проверяет, есть ли в кэше сохраненные и не истёкшие по времени жизни статические данные (HTML, CSS, JS, изображения) и ip адрес сайта для ускорения загрузки  
3. Если в кэше DNS нет ip сайта, обращается на DNS сервер, который возвращает ip адрес сайта  
4. Установка TCP соединения  
5. Установка SSL/TLS-соединения (если HTTPS)  
Client Hello: Клиент отправляет список поддерживаемых протоколов шифрования и случайные данные.  
Server Hello: Сервер выбирает протокол шифрования из предложенных и отправляет свой сертификат SSL/TLS.  
Проверка сертификата: Клиент проверяет подлинность сертификата с помощью доверенного центра сертификации (CA).  
Обмен ключами: Клиент и сервер согласуют секретные ключи для шифрования данных.  
Шифрование установлено: Данные передаются в зашифрованном виде.  
6. Клиент делает http(s) запрос на сервер, сервер отвечает  
7. Рендеринг страницы  
8. Завершение соединения, если в заголовке запроса нет Connection: keep-alive
