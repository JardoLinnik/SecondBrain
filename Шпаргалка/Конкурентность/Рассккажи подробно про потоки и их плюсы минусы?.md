Определение: Поток — это наименьшая единица выполнения внутри процесса. Потоки одного процесса разделяют общую память и ресурсы процесса. В Python потоки создаются с помощью модуля threading. Переключает потоки ОС и GIL.  
  
Плюсы:  
* Легковесность: Создание и переключение между потоками требует меньше ресурсов по сравнению с процессами.  
* Общая память: Потоки могут легко обмениваться данными через общую память, что упрощает коммуникацию.  
* Подходят для I/O-bound задач: Могут улучшить производительность в задачах, связанных с вводом-выводом, за счет перекрытия операций ожидания.  
  
Минусы:  
* Глобальная блокировка интерпретатора (GIL): В CPython GIL позволяет исполнять байткод только одному потоку за раз, что ограничивает параллелизм в CPU-интенсивных задачах.  
* Проблемы с безопасностью потоков: Общая память может привести к гонкам данных и требует использования механизмов синхронизации (блокировки, семафоры).  
* Сложность отладки: Многопоточные приложения сложнее отлаживать из-за параллельного выполнения кода.  
  
Параллельность на уровне ОС  
Потоки переключаются по таймеру ОС (выполняются псевдопараллельно), если они выполняются на одном ядре, но выполнение на одном ядре не обязательно следует из выполнения внутри одного процесса. ОС может раскидать потоки одного процесса по разным ядрам и тогда они будут истинно параллельно выполняться, если оба ядра не заняты другими вычислениями. Можно изменять таймер через модуль sys в python. У каждого потока есть свой приоритет и ОС чаще отдаёт приоритет потокам с бОльшим приоритетом.
