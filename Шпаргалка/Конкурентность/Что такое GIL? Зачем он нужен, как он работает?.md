GIL (Global Interpreter Lock) — это механизм, присутствующий в интерпретаторе CPython, который предотвращает выполнение нескольких потоков одновременно. Это означает, что даже если программа Python создает несколько потоков, только один поток может исполняться в любой момент времени.  
  
Основная причина появления GIL связана с внутренней реализацией CPython и необходимостью управлять памятью (например, подсчет ссылок на объекты) безопасным образом.  
  
GIL работает, обеспечивая эксклюзивный доступ к интерпретатору только одному потоку в каждый момент времени. Он позволяет избежать сложных ошибок синхронизации, но ограничивает параллелизм на уровне потоков  
  
### Решения для обхода ограничения GIL:  
* Использование многопроцессорности, через multiprocessing  
* Использовать альтернативные интерпретаторы, например IronPython  
* Обходится с помощью IO-bound задач и библиотек написанных на С (numpy, pandas, ...)  
* Использовать суб-интерпретаторы  
* В 13-й версии питона можно отключить гил
