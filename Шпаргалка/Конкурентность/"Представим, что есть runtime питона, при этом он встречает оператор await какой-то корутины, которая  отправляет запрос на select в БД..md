В каком месте происходит прерывание? В каком месте event loop отдается контроль? Когда идет продолжение этой задачи?"


1. Вызывается асинхронная функция, которая внутри себя делает await на некую операцию (например, обращение к базе данных).  
2. При достижении оператора await эта корутина «приостанавливается»: Python «сворачивает» контекст выполнения (всё состояние функции), и управление возвращается в event loop.  
3. Event loop в это время может заниматься чем-то ещё: либо переключать другие корутины, либо дожидаться поступления данных (IO) с помощью механизма вроде select/epoll.  
4. Когда результат готов (например, когда драйвер БД сообщает, что данные получены), event loop «пробуждает» корутину и возобновляет её выполнение сразу после строки с await.  
5. Корутина продолжается, используя полученные данные (результат SQL-запроса и т.д.).
