Существуют следующие типы индексов:  
  
Уникальный (Unique index) — все значения встречаются только один раз. Если в столбец с уникальным индексом попробовать добавить неуникальное значение, то возникнет ошибка.Неуникальный (Non-unique index) — индекс, при котором значения могут повторяться.Простой (Simple index) — индекс, состоящий из одного поля.Составной (Composite Index) — индекс, который строится по нескольким столбцам таблицы. В данном типе индекса расположение полей является важным.Частичный (Partial Indexes) — индекс, который состоит из подмножеств строк таблицы по определенному выражению.B-tree (сбалансированное дерево) -  
создается по умолчанию при команде CREATE INDEX, по умолчанию без команды для primary key и unique столбцов.  
- Поддерживает все операторы сравнения  
- Поддерживает LIKE ‘abc%’, но не ‘%abc’  
- Индексирует NULL  
- Умеет возвращать данные в отсортированном виде  
- Сложность поиска О(log N)  
- B-tree используется вместо бинарного дерева поиска (BST) потому что:  
--- в BST не гарантируется сбалансированность, из-за чего сложность поиска может вырасти до О(n)  
--- в BST узел имеет всего два дочерних узла, из-за чего дерево выше -> операций чтения с диска нужно намного большеHash  
CREATE INDEX index_name ON table_name USING HASH (column_name)  
- Поддерживает только оператор =  
- Не всегда, но чаще всего менее выгоден чем B-tree  
- Сложность поиска O(1)  
GiST (обобщенное дерево поиска)  
- для индексации геометрических типов данных и текстовых для полнотекстового поиска.  
- занимает меньше памяти, чем GIN  
- может быть менее эффективен GIN за счет доп проверок  
GIN (обобщенный обратный)  
- для индексации массивов и наборов значения (ключ-значение, диапазоны и др.)  
- для использования более сложного полнотекстового поиска  
SP-GiST (GiST с двоичным разбиением пространства)  
- GIST для несбалансированных наборов данных, но естественно упорядоченных.(например, телефонные номера, разделенные по городам и странам, но основная часть номера рандомная)  
BRIN (блочно-диапазонный)  
- Для естественно-упорядоченных огромных наборов данных (почтовый индекс, timestamp)  
Покрывающий индекс  
- Хранят дополнительные данные, позволяя выполнять запросы без обращения к основной таблице. Делается с помощью ключевого слова INCLUDE() в PostgreSQL
