Нереляционные БД или NoSQL (Not Only SQL) придумали для решения определенных задач, в которых реляционные СУБД показывают себя хуже. К примеру Redis отлично подходит для кеширования, ElasticSearch для организации полнотекстового поиска, а колоночные БД для агрегации запросов на больших объемах данных
