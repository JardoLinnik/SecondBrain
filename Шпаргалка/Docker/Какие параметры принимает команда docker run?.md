-d - запуск контейнера в фоновом режиме  
–rm - удаление контейнера после его остановки  
-p - проброс портов  
–network - выбор сети для контейнера  
–name - присвоить контейнеру заданное имя  
–memory - ограничение объёма оперативной памяти, доступной для контейнера  
-v - монтировать volume  
-e - задать переменные окружения  
-u - запуск контейнера от имени указанного пользователя
