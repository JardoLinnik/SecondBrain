Сервис — это объект, предназначенный для предоставления доступа к приложению, запущенному в виде набора подов в кластере. Он перенаправляет запросы на поды и балансирует нагрузку между ними, разрешает dns имена.  
  
Основные сервисы:  
* Container runtime - отвечает за создание и контроль контейнеров на каждом из узлов.  
* Kubelet - сервис, отвечающий за коммуникацию между узлами  
* kube-proxy - отвечает за сетевые ресурсы узла  
  
Сервисы главного узла:  
* API Server - коммуникация с рабочими узлами  
* Scheduler - балансировщик нагрузки между узлами  
* etcd - сохранение логов о кластере  
* Dns
