Магические методы (dunder-методы) в Python — это специальные методы, которые определяют поведение объекта при использовании встроенных операций языка (например, создание объекта, сложение, вывод на экран и т.д.). Они начинаются и заканчиваются двойным подчеркиванием (например, __init__).  
  
* Методы создания и инициализации:  
__new__(cls, ...): вызывается при создании экземпляра класса (до __init__). Обычно применяется для контроля над процессом выделения памяти или при использовании паттернов проектирования (например, Singleton).  
__init__(self, ...): инициализирует объект сразу после его создания, чаще всего используется для присвоения начальных значений полям.  
  
* Строковое представление  
__repr__(self): возвращает строку, которая однозначно описывает объект. Обычно используется для отладки и должна давать информацию, необходимую для воссоздания объекта.  
__str__(self): возвращает “читаемое” строковое представление объекта (часто более «человеческое», чем __repr__). Вызывается, например, при print(obj).  
  
  
* Сравнение и хеширование  
__eq__(self, other): определяет поведение оператора равенства ==.  
__ne__(self, other): оператор !=.  
__lt__(self, other): оператор <.  
__le__(self, other): оператор <=.  
__gt__(self, other): оператор >.  
__ge__(self, other): оператор >=.  
__hash__(self): возвращает хеш объекта (нужно для использования в set, как ключи в dict и т.д.). Обычно реализуют вместе с __eq__.  
  
* Арифметические операции (перегрузка операторов)  
__add__(self, other): оператор +.  
__radd__(self, other): оператор +, если слева другой тип (reverse add). Аналогично для остальных арифметических методов: __rsub__, __rmul__, и т.д.  
__sub__(self, other), __mul__(self, other), __truediv__(self, other), __floordiv__(self, other) и т. д.: переопределение операций вычитания, умножения, деления и прочего.  
__iadd__(self, other): оператор +=. Аналогично для __isub__ (для -=), __imul__ (для *=) и т.д.  
  
* Преобразование типов  
__bool__(self): вызывается при проверке истинности (if obj: …). Возвращает True или False.  
__int__(self): вызывается при преобразовании к int, например int(obj).  
__float__(self), __complex__(self), __bytes__(self) и т.д. аналогично для соответствующих типов.  
  
* Контейнеры и коллекции  
__len__(self): возвращает длину объекта (например, при вызове len(obj)).  
__getitem__(self, key): доступ к элементам по ключу или индексу (например, obj[key]).  
__setitem__(self, key, value): установка значения по индексу/ключу (например, obj[key] = value).  
__delitem__(self, key): удаление элемента по ключу (например, del obj[key]).  
__contains__(self, item): оператор in (например, item in obj).  
__iter__(self): возвращает итератор или сам объект, если он итерируемый. Позволяет работать в циклах for.  
__next__(self): возвращает следующий элемент итератора, выбрасывает StopIteration, когда элементы заканчиваются.  
  
* Вызов объектов и функции  
__call__(self, ...): делает экземпляр класса вызываемым как функция (например, obj()).  
Контекстный менеджер  
__enter__(self): вызывается при входе в блок with.  
__exit__(self, exc_type, exc_val, exc_tb): вызывается при выходе из блока with (даже при возникновении исключений).  
  
* Работа с атрибутами  
__getattr__(self, name): вызывается при обращении к несуществующему атрибуту (или если обычный механизм поиска атрибута не нашёл нужный).  
__setattr__(self, name, value): вызывается при попытке установить атрибут (даже существующий).  
__delattr__(self, name): вызывается при удалении атрибута.  
  
* Прочие полезные методы  
__dir__(self): вызывается при выполнении dir(obj). Можно кастомизировать список возвращаемых атрибутов.  
__format__(self, format_spec): поддержка формата format(obj, format_spec) и f-строк с форматированием (например, f"{obj:fmt_spec}").  
__index__(self): переопределяет поведение при использовании объекта как индекса в срезах (например, list[obj:]).
