Cборщик мусора в Python реализован с помощью комбинации подсчета ссылок и сборки циклических ссылок. Эти механизмы работают вместе, чтобы автоматически освобождать память, занимаемую объектами, которые больше не используются.  
  
1. Подсчет ссылок  
Подсчет ссылок — это основной механизм управления памятью в Python. Каждому объекту в Python присваивается счетчик ссылок, который показывает, сколько раз на него ссылаются. Когда этот счетчик достигает нуля, объект автоматически удаляется, а занимаемая им память освобождается. Выполняется постоянно.  
  
2. Сборка циклических ссылок (циклический сборщик) (GC)  
Подсчет ссылок не может справиться с циклическими ссылками, то есть ситуацией, когда объекты ссылаются друг на друга, образуя замкнутый круг.  
Для решения этой проблемы Python использует алгоритм отслеживания циклов, который работает параллельно с подсчетом ссылок и периодически ищет циклические зависимости объектов, которые больше не используются (не имеют доступа извне).  
  
Циклический сборщик работает в три этапа (так называемые поколения), чтобы повысить производительность:  
1. Первое поколение: Объекты, которые созданы недавно и могут быть удалены быстро, проверяются первым.  
2. Второе поколение: Объекты, которые пережили первую проверку и, вероятно, будут использоваться дольше.  
3. Третье поколение: Самые долгоживущие объекты, которые проверяются реже всего.  
Эта модель позволяет сократить время работы сборщика мусора, так как не все объекты проверяются каждый раз.  
  
Можно управлять сборщиком мусора через модуль gc (например, отключить его)
