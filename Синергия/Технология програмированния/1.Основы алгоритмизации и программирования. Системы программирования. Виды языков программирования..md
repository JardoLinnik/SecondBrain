Алгоритмизация - создание условий для поэтапного выполнения задачи.

Алгоритм - точное предписание исполнителю совершить последовательность действий, направленных на решение поставленной задачи.

Позиции для составления алгоритма 
1. обозначить кто будет исполнять алгоритм.
2. комбинация команд для исполнителя.
3. форма записи алгоритма.

Характеристики исполнителя: 
- среда
- система команд
- элементарные действия

Свойства алгоритма:
- ясность для исполнителя
- дискретность - разбивка алгоритма на мелкие этапы и последовательное их исполнение.
- детерминированность - каждое правило должно быть четким, однозначным и не оставлять места для двухзначного понимания.
- результативность - алгоритм должен приводить к выполнению поставленной задачи за установленное количество шагов.
- алгоритм решения задачи должен производиться в общем виде

Классификация языков программирования разного уровня.
1. Машинный код - код который воспринимают только компьютеры. Крайне сложный для восприятия человеком.
2. Низкий уровень - языки ассемблера использующие машинные команды закодированные с помощью мнемонических кодов.
3. Средний уровень - могут считаться и низко и высокоуровневыми. например C и C++.
4. Высокий уровень - позволяют создавать сложные алгоритмы , но требуют обработки перед выполнением.
5. Сверхвысокоуровневый - группа имеющая сверхмощные команды и операторы.


Машинный язык - единственны язык понятный ЭВМ. Реализуется аппаратно каждую команду выполняет электронное устройство. Буквально нули и единицы.

Ассе́мблер — транслятор программы из текста на языке ассемблера в программу на машинном языке. Как и сам язык, ассемблеры, как правило, специфичны для конкретной архитектуры, операционной системы и варианта синтаксиса языка, поскольку работают с мнемониками машинных инструкций определённого процессора.

Мнемонический код – это **последовательность символов (букв, цифр, специальных символов и др.), облегчающих запоминание необходимой информации**. Мнемоники бывают цифровыми и символьными.

Языки низкого уровня обеспечивают :
- высокий уровень выполнения программ и максимальную компактность.
- полный контроль над памятью.

Недостатки:
- для каждого типа ЭВМ определенная система.
- сложность и низкая скорость процесса программирования.
- высокая вероятность ошибок
- отсутствие мобильности программ

Отличие языков программирования высокого от низкого уровня:
- использование переменных
- возможность записи сложных приложений
- расширяемость типов данных за счет конструирования новых типов из базовых
- расширяемость набора операций за счет подключения библиотек подпрограмм
- слабая зависимость от типа ЭВМ
